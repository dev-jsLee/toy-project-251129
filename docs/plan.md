# 생산성 플랫폼 기획서

## 1. 프로젝트 개요

### 프로젝트명
**LifeFlow** - 통합 생산성 관리 플랫폼

### 한 줄 소개
할 일 관리를 시작으로 시간 추적, 메모, 습관, 목표, 일정 관리, 종합 분석까지 하나의 플랫폼에서 통합 관리하는 생산성 도구

### 프로젝트 비전 및 목표
- **비전**: 개인의 모든 생산성 활동을 하나의 플랫폼에서 통합 관리하여 더 나은 삶의 흐름을 만들어가는 것
- **목표**: 
  - 단순한 할 일 관리를 넘어 시간, 습관, 목표까지 통합 관리
  - 데이터 기반의 인사이트를 통해 사용자의 생산성 패턴을 이해하고 개선 제안
  - 각 기능이 유기적으로 연결되어 시너지를 만들어내는 생태계 구축

### 타겟 사용자
- **주요 타겟**: 생산성 향상에 관심이 있는 개인 사용자
  - 할 일 관리를 넘어 자신의 시간과 습관을 체계적으로 관리하고 싶은 사람
  - 여러 앱을 오가며 사용하는 것이 번거로워 통합 솔루션을 원하는 사람
  - 자신의 생산성 패턴을 데이터로 확인하고 개선하고 싶은 사람
- **보조 타겟**: 
  - 목표 지향적인 학습자 및 자기계발에 관심이 있는 사람
  - 시간 관리가 중요한 프리랜서 및 원격 근무자

### 핵심 가치 제안
1. **통합성**: 할 일, 시간, 습관, 목표, 일정을 하나의 플랫폼에서 관리
2. **연결성**: 각 기능이 서로 연결되어 데이터가 시너지를 만들어냄
3. **인사이트**: 수집된 데이터를 분석하여 개인화된 생산성 인사이트 제공
4. **점진적 성장**: 할 일 관리부터 시작하여 필요에 따라 기능을 확장해 나가는 구조
5. **데이터 기반**: 추측이 아닌 실제 데이터를 바탕으로 한 생산성 개선

## 2. 핵심 기능 정의 (7단계)

### 2.1 할 일 관리 (MVP)

#### 기본 기능
- **할 일 생성**: 제목, 설명, 마감일 입력
- **할 일 수정**: 내용 변경, 상태 업데이트
- **할 일 삭제**: 완료된 항목 또는 불필요한 항목 제거
- **완료 처리**: 할 일 완료 표시 및 완료 날짜 기록

#### 부가 기능
- **우선순위 설정**: 높음/보통/낮음 또는 1-5점 척도
- **카테고리 분류**: 업무, 개인, 학습, 건강 등 사용자 정의 카테고리
- **마감일 관리**: 날짜 및 시간 설정, 마감일 임박 알림
- **상태 관리**: 대기중/진행중/완료/보류 등 상태 추적
- **하위 할 일**: 큰 할 일을 작은 단위로 분해하여 관리
- **태그 시스템**: 여러 태그를 할 일에 부여하여 유연한 분류

#### 데이터 구조
- 할 일 ID, 제목, 설명, 생성일, 마감일, 완료일
- 우선순위, 카테고리, 태그, 상태
- 부모 할 일 ID (하위 할 일인 경우)
- 연결된 메모, 시간 기록, 습관, 목표 참조

---

### 2.2 메모 기능

#### 할 일 연동 메모
- **상세 메모**: 각 할 일에 상세 정보, 체크리스트, 참고 자료 기록
- **진행 기록**: 할 일 진행 과정에서의 메모 및 업데이트 기록
- **완료 회고**: 할 일 완료 후 느낀 점, 배운 점 기록

#### 독립 메모 노트
- **일반 메모**: 할 일과 무관한 아이디어, 생각, 정보 기록
- **회의록/학습 노트**: 업무 회의나 학습 내용 기록
- **프로젝트 노트**: 특정 프로젝트 관련 종합 정보 관리

#### 검색 및 태그 기능
- **전문 검색**: 제목, 내용, 태그를 통한 검색
- **태그 관리**: 태그 생성, 수정, 삭제 및 태그별 메모 필터링
- **날짜 필터**: 특정 기간의 메모 조회
- **연결된 할 일 표시**: 메모와 연결된 할 일 목록 표시

#### 데이터 구조
- 메모 ID, 제목, 내용, 생성일, 수정일
- 태그 목록
- 연결된 할 일 ID (있는 경우)
- 메모 타입 (할 일 연동 / 독립)

---

### 2.3 시간 추적

#### 할 일별 시간 기록 방법
- **수동 입력**: 할 일에 소요된 시간을 직접 입력
- **타이머 기록**: 실시간 타이머로 작업 시간 측정
- **자동 추정**: 과거 데이터 기반 소요 시간 추정 제안

#### 타이머 기능
- **시작/일시정지/정지**: 할 일별 타이머 제어
- **백그라운드 실행**: 다른 화면으로 이동해도 타이머 유지
- **알림 기능**: 설정한 시간 경과 시 알림
- **다중 타이머**: 여러 할 일의 타이머를 동시에 관리 (활성화된 것만 측정)

#### 시간 통계 및 리포트
- **일별 통계**: 하루 동안 각 할 일에 소요된 시간
- **주별/월별 리포트**: 기간별 시간 사용 패턴 분석
- **카테고리별 분석**: 카테고리별 시간 분배 현황
- **생산성 시간대**: 가장 생산적인 시간대 파악
- **시간 트렌드**: 시간 사용 패턴의 변화 추이

#### 데이터 구조
- 시간 기록 ID, 할 일 ID
- 시작 시간, 종료 시간, 소요 시간
- 기록 타입 (수동 입력 / 타이머)
- 작업 날짜

---

### 2.4 습관 추적

#### 반복 할 일을 습관으로 전환
- **습관 생성**: 반복되는 할 일을 습관으로 등록
- **반복 주기 설정**: 매일, 매주, 매월 등 반복 주기 선택
- **요일별 설정**: 특정 요일에만 실행되는 습관 설정

#### 습관 체인 및 연속 기록
- **체인 표시**: 연속으로 수행한 일수 시각화
- **최장 연속 기록**: 개인 최고 기록 추적
- **체인 유지 알림**: 체인이 끊어지기 전 알림
- **체인 복구**: 실수로 체인이 끊어진 경우 복구 기능

#### 습관별 통계
- **완료율**: 설정한 기간 동안의 습관 완료율
- **연속 기록 통계**: 평균 연속 일수, 최장 연속 일수
- **습관 히트맵**: 달력 형태로 습관 수행 여부 시각화
- **습관별 시간 분석**: 각 습관에 소요된 시간 통계

#### 데이터 구조
- 습관 ID, 이름, 설명
- 반복 주기, 요일 설정
- 시작일, 목표일 (있는 경우)
- 연결된 할 일 ID 목록
- 완료 기록 (날짜별 완료 여부)
- 연속 기록 일수, 최장 연속 기록

---

### 2.5 목표 관리

#### 목표 생성 및 구조화
- **목표 생성**: 장기/단기 목표 생성
- **목표 분해**: 큰 목표를 작은 목표로 분해
- **목표 계층 구조**: 목표-하위목표-할 일의 3단계 구조
- **목표 기간 설정**: 목표 달성 기한 설정

#### 할 일과 목표 연결
- **할 일 할당**: 기존 할 일을 목표에 연결
- **목표별 할 일 생성**: 목표에서 직접 할 일 생성
- **목표별 할 일 필터링**: 특정 목표와 관련된 할 일만 조회

#### 진행률 추적
- **자동 진행률 계산**: 연결된 할 일 완료율 기반 진행률 계산
- **수동 진행률 설정**: 할 일과 무관한 목표의 경우 수동 설정
- **목표별 통계**: 목표 달성을 위한 시간 투자, 완료한 할 일 수 등
- **목표 달성 알림**: 목표 달성 시 축하 메시지 및 다음 목표 제안

#### 데이터 구조
- 목표 ID, 제목, 설명
- 목표 타입 (장기/단기), 목표 기간
- 부모 목표 ID (하위 목표인 경우)
- 연결된 할 일 ID 목록
- 진행률, 목표 달성일

---

### 2.6 일정 관리

#### 할 일을 캘린더로 변환
- **일정 변환**: 할 일을 특정 날짜/시간의 일정으로 변환
- **일정 생성**: 할 일과 독립적으로 일정 생성
- **일정 수정**: 시간 조정, 일정 이동
- **일정 삭제**: 불필요한 일정 제거

#### 시간 블로킹
- **시간 블록 생성**: 특정 시간대를 특정 작업으로 블록 지정
- **할 일 기반 블록**: 할 일의 예상 소요 시간을 고려한 블록 제안
- **블록 충돌 체크**: 이미 예약된 시간과의 충돌 감지 및 알림
- **블록 이동**: 드래그 앤 드롭으로 블록 시간 조정

#### 일정 충돌 관리
- **충돌 감지**: 일정 간 시간 겹침 자동 감지
- **충돌 해결 제안**: 충돌 발생 시 해결 방안 제안
- **우선순위 기반 정렬**: 우선순위가 높은 일정 우선 표시

#### 캘린더 뷰
- **월간 뷰**: 한 달 전체 일정 조회
- **주간 뷰**: 주 단위 상세 일정 조회
- **일간 뷰**: 하루 단위 시간대별 일정 조회
- **할 일과 일정 통합 표시**: 할 일과 일정을 함께 표시

#### 데이터 구조
- 일정 ID, 제목, 설명
- 시작 시간, 종료 시간
- 연결된 할 일 ID (있는 경우)
- 반복 설정 (있는 경우)
- 알림 설정

---

### 2.7 종합 분석 대시보드

#### 생산성 지표 정의
- **완료율**: 설정한 할 일 대비 완료율
- **시간 효율성**: 계획한 시간 대비 실제 소요 시간
- **습관 일관성**: 습관 완료율 및 연속 기록
- **목표 진행도**: 목표별 진행률 및 달성률
- **생산성 점수**: 종합적인 생산성 지표 (0-100점)

#### 통합 통계 및 시각화
- **시간 사용 분석**: 카테고리별, 할 일별 시간 분배 차트
- **완료 추이**: 일별/주별/월별 할 일 완료 추이 그래프
- **습관 히트맵**: 습관 수행 현황 달력 히트맵
- **목표 진행 차트**: 목표별 진행률 및 달성 현황
- **생산성 시간대**: 시간대별 생산성 분석 그래프
- **주간/월간 리포트**: 기간별 종합 리포트 생성

#### 인사이트 및 개선 제안
- **패턴 분석**: 사용자의 생산성 패턴 자동 분석
- **개선 제안**: 데이터 기반 개인화된 개선 제안
  - 가장 생산적인 시간대에 중요한 할 일 배치 제안
  - 습관 완료율이 낮은 경우 주기 조정 제안
  - 시간 추정이 부정확한 경우 더 현실적인 시간 계획 제안
- **성과 요약**: 주간/월간 성과 요약 및 하이라이트
- **트렌드 알림**: 생산성 변화 트렌드 및 경고

#### 데이터 구조
- 사용자 ID
- 분석 기간
- 각 기능별 통계 데이터 집계
- 생산성 지표 값
- 인사이트 및 제안 텍스트

## 3. 사용자 시나리오 및 사용 흐름

### 3.1 주요 사용 시나리오

#### 시나리오 1: 새로운 할 일 추가 및 관리
1. 사용자가 "프로젝트 기획서 작성" 할 일을 생성
2. 마감일, 우선순위, 카테고리 설정
3. 할 일에 상세 메모 추가 (체크리스트 포함)
4. 작업 시작 시 타이머 시작
5. 작업 완료 후 타이머 정지 및 완료 처리
6. 완료 회고 메모 작성

#### 시나리오 2: 습관 형성 및 추적
1. "매일 운동하기" 할 일을 습관으로 전환
2. 매일 반복 설정 및 알림 시간 설정
3. 매일 운동 완료 시 체크
4. 연속 기록이 쌓이면서 체인 시각화 확인
5. 주간 습관 통계 확인 및 동기 부여

#### 시나리오 3: 목표 설정 및 달성 추적
1. "6개월 내 영어 회화 실력 향상" 장기 목표 생성
2. 목표를 "단어 암기", "회화 연습", "영화 시청" 등 하위 목표로 분해
3. 각 하위 목표에 할 일 연결
4. 할 일 완료 시 자동으로 목표 진행률 업데이트
5. 목표 달성 시 축하 메시지 및 다음 목표 제안

#### 시나리오 4: 시간 블로킹 및 일정 관리
1. 오늘 해야 할 할 일 목록 확인
2. 각 할 일의 예상 소요 시간 확인
3. 할 일을 캘린더의 특정 시간대로 배치
4. 시간 블록 생성 및 충돌 체크
5. 실제 작업 시 타이머로 시간 측정
6. 계획과 실제 소요 시간 비교

#### 시나리오 5: 생산성 분석 및 개선
1. 주간 리포트 확인
2. 시간 사용 패턴 분석 (어떤 카테고리에 시간을 많이 쓰는지)
3. 생산성 시간대 확인 (언제 가장 집중하는지)
4. 습관 완료율 및 연속 기록 확인
5. AI 인사이트를 통한 개선 제안 확인
6. 제안에 따라 다음 주 계획 조정

### 3.2 기능 간 연결 흐름

```
할 일 관리 (시작점)
    ↓
메모 기능 ← 할 일에 상세 정보 추가
    ↓
시간 추적 ← 할 일 작업 시간 측정
    ↓
습관 추적 ← 반복 할 일을 습관으로 전환
    ↓
목표 관리 ← 할 일을 목표에 연결
    ↓
일정 관리 ← 할 일을 캘린더로 변환
    ↓
종합 분석 ← 모든 데이터 통합 분석
```

### 3.3 사용자 여정 맵

**초기 사용자 (1주차)**
- 할 일 추가 및 완료 처리
- 기본적인 할 일 관리에 익숙해짐

**성장 사용자 (2-4주차)**
- 메모 기능 활용 시작
- 시간 추적으로 자신의 작업 패턴 파악
- 습관 기능으로 반복 작업 관리

**활발 사용자 (1-3개월)**
- 목표 설정 및 추적
- 일정 관리로 시간 계획
- 주간 리포트 확인

**고급 사용자 (3개월 이상)**
- 종합 분석 대시보드 적극 활용
- 인사이트 기반 생산성 개선
- 모든 기능을 통합적으로 활용

## 4. 정보 구조 및 데이터 관계

### 4.1 핵심 엔티티 정의

#### 할 일 (Task)
- 기본 정보: ID, 제목, 설명, 생성일, 마감일, 완료일
- 분류: 우선순위, 카테고리, 태그, 상태
- 관계: 부모 할 일, 연결된 메모, 시간 기록, 습관, 목표, 일정

#### 메모 (Note)
- 기본 정보: ID, 제목, 내용, 생성일, 수정일
- 분류: 태그, 메모 타입
- 관계: 연결된 할 일

#### 시간 기록 (TimeEntry)
- 기본 정보: ID, 시작 시간, 종료 시간, 소요 시간
- 관계: 연결된 할 일

#### 습관 (Habit)
- 기본 정보: ID, 이름, 설명, 시작일, 목표일
- 설정: 반복 주기, 요일 설정
- 기록: 완료 기록, 연속 기록 일수, 최장 연속 기록
- 관계: 연결된 할 일 목록

#### 목표 (Goal)
- 기본 정보: ID, 제목, 설명, 목표 기간
- 진행: 진행률, 목표 달성일
- 관계: 부모 목표, 하위 목표, 연결된 할 일 목록

#### 일정 (Event)
- 기본 정보: ID, 제목, 설명
- 시간: 시작 시간, 종료 시간
- 설정: 반복 설정, 알림 설정
- 관계: 연결된 할 일

#### 사용자 (User)
- 기본 정보: ID, 이름, 이메일
- 설정: 사용자 설정, 알림 설정

### 4.2 엔티티 간 관계도

```
User
  ├── Task (1:N)
  │   ├── Note (1:1 또는 1:N)
  │   ├── TimeEntry (1:N)
  │   ├── Habit (N:1)
  │   ├── Goal (N:1)
  │   └── Event (1:1)
  │
  ├── Habit (1:N)
  │   └── Task (1:N)
  │
  ├── Goal (1:N)
  │   ├── Goal (1:N, 하위 목표)
  │   └── Task (1:N)
  │
  └── Event (1:N)
      └── Task (1:1, 선택적)
```

### 4.3 데이터 흐름도

**할 일 생성 흐름**
1. 사용자 입력 → Task 생성
2. Task 생성 시 User와 연결
3. 선택적으로 Note, Habit, Goal, Event와 연결

**시간 추적 흐름**
1. Task 선택 → Timer 시작
2. Timer 종료 → TimeEntry 생성 및 Task와 연결
3. TimeEntry 데이터 → 통계 집계 → 분석 대시보드

**습관 완료 흐름**
1. Habit의 Task 완료 → Habit 완료 기록 업데이트
2. 연속 기록 계산 → Habit 통계 업데이트
3. Habit 데이터 → 분석 대시보드

**목표 진행 흐름**
1. Goal의 Task 완료 → Goal 진행률 자동 계산
2. Goal 진행률 업데이트 → 분석 대시보드

**종합 분석 흐름**
1. 모든 엔티티 데이터 수집
2. 기간별 집계 및 통계 계산
3. 패턴 분석 및 인사이트 생성
4. 대시보드에 시각화하여 표시

## 5. UI/UX 요구사항 (기능 중심)

### 5.1 각 화면의 주요 기능 요소

#### 메인 대시보드
- 오늘의 할 일 목록 (우선순위별 정렬)
- 진행 중인 타이머 (있는 경우)
- 오늘 완료해야 할 습관 체크리스트
- 빠른 액션: 할 일 추가, 타이머 시작
- 생산성 지표 요약 (완료율, 오늘 소요 시간 등)

#### 할 일 관리 화면
- 할 일 목록 (필터: 전체/진행중/완료/보류)
- 할 일 카드: 제목, 마감일, 우선순위, 카테고리, 상태 표시
- 할 일 상세: 전체 정보, 연결된 메모, 시간 기록, 관련 목표 표시
- 할 일 추가/수정 폼

- 메모 목록 (할 일 연동 / 독립 메모 구분)

- 태그 필터
- 메모 에디터 (제목, 내용, 태그 입력)
- 연결된 할 일 표시

#### 시간 추적 화면
- 활성 타이머 표시 (큰 타이머 UI)
- 할 일별 시간 기록 목록
- 시간 통계 차트 (일별/주별/월별)
- 카테고리별 시간 분배 파이 차트
- 생산성 시간대 그래프

#### 습관 화면
- 습관 목록 (체인 시각화 포함)
- 습관 카드: 이름, 연속 기록 일수, 완료율
- 습관 상세: 히트맵, 통계, 연결된 할 일
- 습관 추가/수정 폼
- 오늘 완료해야 할 습관 체크리스트

#### 목표 화면
- 목표 목록 (계층 구조 표시)
- 목표 카드: 제목, 진행률, 목표 기간, 연결된 할 일 수
- 목표 상세: 진행률 차트, 하위 목표, 할 일 목록
- 목표 추가/수정 폼
- 목표 달성 축하 화면

#### 일정 관리 화면
- 캘린더 뷰 (월간/주간/일간 전환)
- 일정 블록 표시 (시간대별)
- 할 일을 일정으로 변환 버튼
- 일정 추가/수정/삭제
- 충돌 알림 표시

#### 종합 분석 대시보드
- 생산성 점수 (큰 숫자로 표시)
- 주요 지표 카드 (완료율, 시간 효율성, 습관 일관성 등)
- 시간 사용 분석 차트
- 완료 추이 그래프
- 습관 히트맵
- 목표 진행 차트
- 인사이트 섹션 (개선 제안)
- 리포트 다운로드 기능

## 7. 차별화 포인트

### 7.1 기존 할 일 관리 앱과의 차별점

#### 통합성
- **기존 앱**: 할 일 관리만 하거나, 다른 기능이 있어도 분리되어 있음
- **LifeFlow**: 할 일, 시간, 습관, 목표, 일정이 하나의 플랫폼에서 유기적으로 연결됨

#### 데이터 시너지
- **기존 앱**: 각 기능이 독립적으로 동작하여 데이터가 시너지를 만들지 못함
- **LifeFlow**: 할 일 완료가 자동으로 습관 체인, 목표 진행률, 시간 통계에 반영되어 전체적인 생산성 향상에 기여

#### 점진적 성장
- **기존 앱**: 처음부터 모든 기능을 제공하여 복잡함
- **LifeFlow**: 할 일 관리부터 시작하여 필요에 따라 기능을 추가해 나가는 자연스러운 성장 경험

#### 인사이트 제공
- **기존 앱**: 단순한 통계만 제공
- **LifeFlow**: 데이터 기반의 개인화된 인사이트와 개선 제안 제공

### 7.2 통합 생산성 플랫폼의 장점

#### 일관된 사용자 경험
- 하나의 앱에서 모든 생산성 활동을 관리하여 앱 간 전환 불필요
- 통일된 디자인과 인터페이스로 학습 곡선 최소화

#### 데이터의 통합 분석
- 할 일, 시간, 습관, 목표 데이터를 종합하여 더 깊이 있는 분석 가능
- 예: "이 목표를 위해 얼마나 시간을 투자했는지" 같은 통합 질문에 답변 가능

#### 자동화된 연동
- 할 일 완료가 자동으로 습관 체인, 목표 진행률에 반영
- 수동 동기화 불필요

#### 생산성 생태계
- 각 기능이 서로 보완하여 전체적인 생산성 향상
- 예: 시간 추적 데이터로 일정 관리 시 더 현실적인 시간 블로킹 가능

### 7.3 사용자에게 제공하는 가치

#### 시간 절약
- 여러 앱을 오가며 사용할 필요 없음
- 자동 연동으로 수동 작업 최소화

#### 자기 이해
- 데이터 기반으로 자신의 생산성 패턴을 이해
- 언제 가장 생산적인지, 어떤 습관이 효과적인지 파악

#### 동기 부여
- 습관 체인, 목표 진행률 등 시각적 피드백으로 동기 부여
- 성과를 데이터로 확인하여 성취감 제공

#### 지속적인 개선
- 인사이트를 통한 개선 제안으로 생산성 지속 향상
- 트렌드 분석으로 장기적인 변화 추적

#### 목표 달성
- 할 일을 목표에 연결하여 목표 달성 과정을 체계적으로 관리
- 진행률 추적으로 목표 달성 가능성 파악

- 프로젝트별 예산 관리

### 8.2 확장 시나리오

#### 시나리오 1: 개인 사용자 → 팀 사용자
- 현재: 개인 생산성 관리
- 확장: 팀 협업 기능 추가로 소규모 팀에서 사용

#### 시나리오 2: 기본 기능 → AI 강화
- 현재: 사용자 입력 기반
- 확장: AI가 자동으로 할 일 분류, 일정 제안, 인사이트 생성

#### 시나리오 3: 웹 앱 → 모바일 앱
- 현재: 웹 브라우저에서 사용
- 확장: 모바일 네이티브 앱으로 확장하여 언제 어디서나 접근

#### 시나리오 4: 독립 앱 → 통합 플랫폼
- 현재: 독립적인 생산성 앱
- 확장: 다른 서비스와 연동하여 생산성 생태계의 중심이 됨

#### 시나리오 5: 무료 → 프리미엄
- 현재: 기본 기능 무료 제공
- 확장: 고급 분석, AI 기능, 협업 기능 등을 프리미엄 기능으로 제공

## 결론

LifeFlow는 단순한 할 일 관리 앱을 넘어 사용자의 모든 생산성 활동을 통합 관리하는 플랫폼입니다. 할 일 관리부터 시작하여 점진적으로 기능을 확장해 나가는 구조로, 사용자가 자신의 페이스에 맞춰 기능을 익히고 활용할 수 있습니다.

각 기능이 유기적으로 연결되어 데이터가 시너지를 만들어내고, 종합 분석을 통해 사용자에게 개인화된 인사이트를 제공합니다. 이를 통해 사용자는 자신의 생산성 패턴을 이해하고 지속적으로 개선해 나갈 수 있습니다.
