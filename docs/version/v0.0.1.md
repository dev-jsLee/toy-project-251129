## 버그 수정

### 1. SQLite 데이터베이스 경로 문제 해결
**문제**: 백엔드 실행 시 `sqlite3.OperationalError: unable to open database file` 오류 발생

**원인**: 
- 상대 경로 `sqlite:///db/data.db`가 Windows 환경에서 올바르게 해석되지 않음
- `db` 디렉토리가 존재하지 않을 수 있음

**해결**:
- `backend/config.py`에서 절대 경로 사용으로 변경
- `pathlib.Path`를 사용하여 플랫폼 독립적인 경로 처리
- Windows 경로의 백슬래시를 슬래시로 변환하여 SQLite URI 형식에 맞춤
- `db` 디렉토리 자동 생성 로직 추가

**수정 파일**: `backend/config.py`

**변경 내용**:
```python
# 변경 전
SQLALCHEMY_DATABASE_URI = 'sqlite:///db/data.db'

# 변경 후
BASE_DIR = Path(__file__).resolve().parent
DB_DIR = BASE_DIR / 'db'
DB_DIR.mkdir(exist_ok=True)
DB_PATH = DB_DIR / 'data.db'
db_path = str(DB_PATH).replace('\\', '/')
SQLALCHEMY_DATABASE_URI = f'sqlite:///{db_path}'
```

### 2. 프론트엔드 API 연결 문제 해결
**문제**: 프론트엔드에서 할 일 목록을 불러올 수 없음. `Unknown error` 발생

**원인**:
- 프론트엔드가 `localhost:8000`에서 실행되는데, API 요청이 상대 경로(`/api/v1`)로 전송됨
- 결과적으로 `localhost:8000/api/v1/tasks`로 요청이 가지만, 백엔드는 `localhost:5000`에서 실행 중
- CORS는 설정되어 있었으나 요청이 잘못된 주소로 전송됨

**해결**:
- 개발 환경에서는 절대 URL(`http://localhost:5000/api/v1`) 사용
- 프로덕션 환경(Docker)에서는 상대 URL(`/api/v1`) 사용 (Nginx가 프록시)
- 환경에 따라 자동으로 URL 선택하도록 로직 추가
- 에러 메시지 개선으로 네트워크 오류 시 더 자세한 정보 제공

**수정 파일**: `frontend/js/api/taskApi.js`

**변경 내용**:
```javascript
// 변경 전
const API_BASE_URL = '/api/v1';

// 변경 후
const API_BASE_URL = window.location.hostname === 'localhost' && window.location.port !== '80'
    ? 'http://localhost:5000/api/v1'  // 개발 환경
    : '/api/v1';  // 프로덕션 환경 (Docker)
```

**추가 개선**:
- 네트워크 오류 시 더 자세한 에러 메시지 제공
- 백엔드 서버 연결 실패 시 명확한 안내 메시지 표시
